DHCP - Dynamic Host Configuration Protocol, 
упрощает настройку большого количества клиентов.
Выдаются следующие настройки:
-IP-адресс/маска
-Шлюз
-Домен
-Список DNS-серверов
- и т.д.

Варианты присвоения адреса клиенту DHCP-сервером
Статическое: 
    Адрес назначается администратором вручную
    IP-адерс закрепляется за определенным MAC-адресом
Динамическое:
    Адрес выдается на некоторое время
    Может меняться каждый раз
Автоматическое:
    IP адрес закрепляется за MAC-адресом(не админом, а сервером)
    Выдается навсегда

Обнаружение сервера: DISCOVER
    Источник:
        MAC клиента
        IP - 0.0.0.0
    Получатель
        MAC - FF:FF:FF:FF:FF:FF
        IP - 255.255.255.255
Предложение от сервера: OFFER
    Источник
        MAC - Предполагаемый IP клиента
        IP - IP сервера
    Получатель 
        MAC - MAC клиента
        IP - MAC сервера
    Также сервер отправляет предварительные настройки
Запрос (уведомление) клиента REQUEST
    Подтверждает предложившему серверу, что согласен с настройками
    Сервер помечает IP как занятый
    Отправляется всем в сети, чтобы уведомить все DHCP-серверы
Подтверждение от сервера: ACKNOWLADGEMENT
    Уведомление клиента о резервировании адреса
    Время резервирования адреса

Порты DHCP(IPv4/DHCPv4):
UDP порт 67 - для DHCP-сервера(прослушивает запросы от клиентов)
UDP порт 68 - для DHCP-клиента(принимает ответы от сервера)

Свойство Stateless HTTP означает что каждое последующее
соединение ничего не знает о предыдущем.

10 Объектов браузера:
1. Window - корневой объект
2. document - содержимое веб-страницы через DOM
3. navigator - Сведения о браузере, языке, ОС и т.д.
4. location
5. history - история переходов (history.back(), .forward(), .go(-2))
6. screen - данные об экране пользователя
7. localStorage - Хранилище данных в браузере без срока годности
8. sessionStorage - как localStorage, но данные хранятся в течение сесси
9. console - для вывода отладочной информации в консоль
10. perfomance - данные о производительности загрузки и выполнения страницы
